# Website QA Review Skill

## Objective

Perform a complete pre-delivery QA review to ensure the website is accurate, consistent, secure, and production-ready.

---

# Before You Start

**Do NOT begin the review immediately.**

First, ask for the project's approved **Client Profile**, **Project Story**, or any document containing the official business information.

If no reference exists, stop and request it before continuing.

Required information:

- Company name
- Brand description
- Logo
- Favicon
- Email(s)
- Phone number(s)
- WhatsApp
- Address
- Google Maps URL
- Business hours
- Country & City
- Domain
- Social media accounts
- Brand assets
- Approved content
- Design reference (if available)

**Never guess business information. Always compare the website against the approved reference.**

---

# Company Information

- Verify all business information matches the approved reference.
- Ensure consistency across all pages.

---

# Contact Information

- Phone numbers use `tel:`.
- Emails use `mailto:`.
- WhatsApp opens correctly.
- Google Maps opens the correct location.
- All contact information is clickable.

---

# Social Media

- Every account exists in the approved Client Profile.
- Every link points to the official account.
- No broken or outdated links.
- External links open safely.

---

# Navigation

- No broken links.
- No missing pages.
- Logo returns to Home.
- Navigation is consistent.

---

# Content

- No placeholder content.
- No Lorem Ipsum.
- No TODO or demo text.
- Grammar and spelling are correct.
- Terminology is consistent.
- Avoid ambiguous or misleading wording.

---

# Media

- Images load correctly.
- Icons are consistent.
- Logo is correct.
- Favicon is present.
- Alt text exists.
- No missing assets.

---

# Forms

- Validation works.
- Required fields are enforced.
- Success and error states exist.
- Duplicate submissions are prevented.

---

# Responsive Design

- Mobile
- Tablet
- Desktop

---

# Accessibility

- Keyboard navigation.
- Visible focus states.
- Proper color contrast.
- ARIA labels where needed.

---

# SEO

- Title
- Meta Description
- Canonical
- Open Graph
- Sitemap
- Robots
- Structured Data

---

# Performance

- Images optimized.
- Lazy loading enabled.
- Unused assets removed.
- Console logs removed.
- Production build optimized.

---

# Security

- No secrets in source code.
- Environment variables used correctly.
- Input validation.
- XSS protection.
- Injection protection.
- HTTPS enabled.

# Authentication
